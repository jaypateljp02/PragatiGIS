# FRA Atlas Platform - Complete Feature Enhancement Plan

Based on the current mockups analysis, here's what needs to be added to create a complete solution for both Ministry and State users.

## 1. User Management & Multi-Tenancy

### A. Authentication & Authorization
- **Login Page**: Role-based login (Ministry/State/District/Village)
- **User Roles**:
  - Ministry Admin: Full access to all states
  - State Admin: Access to their state only
  - District Officer: Access to their district only
  - Village Officer: Access to their village only

### B. State-Specific Dashboards
Create separate dashboards for each target state:

**Madhya Pradesh Dashboard**
- URL: `/dashboard/madhya-pradesh`
- District-wise breakdown (Mandla, Balaghat, Dindori, etc.)
- State-specific metrics and KPIs
- Local language support (Hindi)

**Odisha Dashboard** 
- URL: `/dashboard/odisha`
- District-wise breakdown (Mayurbhanj, Keonjhar, etc.)
- Community rights focus
- Local language support (Odia)

**Telangana Dashboard**
- URL: `/dashboard/telangana` 
- District-wise breakdown (Adilabad, Warangal, etc.)
- High pending claims focus
- Local language support (Telugu)

**Tripura Dashboard**
- URL: `/dashboard/tripura`
- District-wise breakdown (West Tripura, South Tripura, etc.)
- Community success model
- Local language support (Bengali)

## 3. Technical Architecture Enhancements

### A. Database Schema Updates
```sql
-- User management tables
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    username VARCHAR(100) UNIQUE,
    email VARCHAR(255),
    role VARCHAR(50), -- 'ministry', 'state', 'district', 'village'
    state_id INTEGER REFERENCES states(id),
    district_id INTEGER REFERENCES districts(id),
    created_at TIMESTAMP DEFAULT NOW()
);

-- Session management
CREATE TABLE user_sessions (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id),
    token VARCHAR(255),
    expires_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT NOW()
);

-- Audit trail
CREATE TABLE audit_log (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id),
    action VARCHAR(100),
    resource_type VARCHAR(50),
    resource_id INTEGER,
    changes JSONB,
    created_at TIMESTAMP DEFAULT NOW()
);
```
